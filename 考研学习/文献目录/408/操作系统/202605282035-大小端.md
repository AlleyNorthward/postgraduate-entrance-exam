# 描述
- 计算机中, 内存和值之间, 总是有种微妙的关系
- 例如地址`0x7c00`, 这是`32`位机器
	- 那么改地址存放的值是`0x12`, 十六进制的两位, 二进制八位, 没有问题
	- 对于给定的地址, 总能放入两位十六进制数(一字节)
- 可是如果给出四位十六进制数, 八位十六进制数, 十六位十六进制数, 应该从哪端放入呢?
	- 在汇编中, `.byte`是一字节, `32`位机器中, `.word`一般是两字节, `.long`是四字节, `.quad`是八字节
	- 那么对于`.byte`, 不用区分, 是多少就是多少, 不用转换
	- `.word`需要转换, 例如`0x1234`, 那么低地址值是`0x34`, 高地址值是`0x12`, 这是小端. 如果内存是`0x2222`, `0x2223`那么`0x2222`值是`0x34`, `0x2223`值是`0x12`
	- 同理, 对于`.quad`, 如果是`0x00cf9a000000ffff`, 那么从低地址到高地址的值, 就很明确
~~~C
struct {
  u16 limit_l;
  u16 base_l;
  u16 basehl_attr;
  u16 base_limit;
} gdt_table[256] __attribute__((aligned(8))) = {

  [KERNEL_CODE_SEG / 8] = {
    0xffff,
    0x0000,
    0x9a00,
    0x00cf
  },

  [KERNEL_DATA_SEG / 8] = {
    0xffff,
    0x0000,
    0x9200,
    0x00cf
  }
};
~~~

- 例如, 结构体中, 每个描述符是八字节, 默认是从低地址到高地址排列, 那么在汇编中的话, 就是这么写`.quad 0x00cf9a000000ffff`, 注意区别
