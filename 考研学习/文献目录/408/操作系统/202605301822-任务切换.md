- 单核cpu的任务切换, 最可能的实现方式是时间片轮转. 可是最简单的时间片轮转机制是什么呢? 定时器中断
- 任务切换需要保护cpu执行的上下文, 怎么保护? tss表, 形式如下
~~~C
uint32_t task0_tss[] = {
    // prelink, esp0, ss0, esp1, ss1, esp2, ss2
    0,  (uint32_t)task0_dpl0_stack + 4*1024, KERNEL_DATA_SEG , /* 后边不用使用 */ 0x0, 0x0, 0x0, 0x0,
    // cr3, eip, eflags, eax, ecx, edx, ebx, esp, ebp, esi, edi,
    (uint32_t)pg_dir,  (uint32_t)task_0/*入口地址*/, 0x202, 0xa, 0xc, 0xd, 0xb, (uint32_t)task0_dpl3_stack + 4*1024/* 栈 */, 0x1, 0x2, 0x3,
    // es, cs, ss, ds, fs, gs, ldt, iomap
    APP_DATA_SEG, APP_CODE_SEG, APP_DATA_SEG, APP_DATA_SEG, APP_DATA_SEG, APP_DATA_SEG, 0x0, 0x0,
};
~~~
- 实际上就是保护了各个寄存器的值
- 除此之外, 还有一个tr寄存器, 专门用于存储当前执行任务
- 这样, 我们可以在定时器中断函数中执行任务切换函数, 即可实现任务切换
~~~C
void task_sched (void) {
    static int task_tss = TASK0_TSS_SEL;

    // 更换当前任务的tss，然后切换过去
    task_tss = (task_tss == TASK0_TSS_SEL) ? TASK1_TSS_SEL : TASK0_TSS_SEL;
    uint32_t addr[] = {0, task_tss };
    __asm__ __volatile__("ljmpl *(%[a])"::[a]"r"(addr));
}
~~~
- 那么任务切换, 到底发生了什么呢? 如何实现的?
- 假设一种情况, 任务A切换到任务B
	- 此时需要保护任务A的执行上下文, 同时需要回复任务B的执行上下文, 所以这一种情况就可以覆盖大部分情况
	- 指令ljmp, 可以直接跳转到任务B, 继续执行
	- 继续分析之前, 需要有几个关键结构, TR寄存器, GDT表, TSS表
	- TR寄存器指向当前执行任务
	- CPU会自动写TSS, 关键是TSS在哪
	- 肯定在内存中, 怎么寻址呢? 通过GDT表, 也就是跟任务A相关的GDT表, 这样通过描述符就能寻址了, CPU也就直接写了
	- 保存完之后, TR寄存器会指向下一任务, 并且CPU能够通过描述符查到TSS表, 并且自动恢复信息
	- 所以, [[202605281535-保护模式|保护模式]]下, 执行函数跳转时, 通过描述符即可完成跳转, 而无需携带其它复杂信息
