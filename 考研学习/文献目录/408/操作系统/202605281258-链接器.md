- 这部分是自定义段, 对于理解编译原理, 内存相关的, 还是很有帮助的
- 编译器层面的段, 有输入段和输出段之说
	- 输入段是在代码中(`.c`或`.s`中显式声明的)
	- 输出段则是链接器层面输出的段, 是对内存的一种抽象
- 一般编译器默认段有`.bss .data .text .rodata`段等, 用户可以在代码中自定义输入段, 也可以在链接代码中自定义输出段
~~~c
ENTRY(_start)

SECTIONS{
  . = 0x7c00;
  .boot :                 // 输出段
  {
    KEEP(*(.boot))        // 输入段
  }

  ASSERT(SIZEOF(.boot) == 512, "boot sector must be exactly 512 bytes")

  . = 0x7e00;
  .protect32 :            // 输出段
  {
    KEEP(*(.text32))      // 输入段
    *(.text*)             // 输入段
  }

  .rodata :
  {
    *(.rodata*)
  }

  .data :
  {
    *(.data*)
  }

  .bss :
  {
    *(.bss*)
    *(COMMON)
  }

  /DISCARD/ :
  {
    *(.eh_frame)
    *(.comment)
    *(.note*)
    *(.gnu.property)
  }
}
~~~
