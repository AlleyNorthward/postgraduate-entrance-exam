#### 自己模仿一下TRY CATCH THROW 机制吧

#include <setjmp.h>
#include <stdio.h>

jmp_buf b;

#define TRY if (setjmp(b) == 0)
#define CATCH else
#define THROW(b) longjmp(b, 1);

int main() {
  TRY {
    printf("int TRY\n");
    THROW(b);
  }
  CATCH { printf("CATCH!\n"); }
}
~~~

#### lua的异常机制并不是TRY CATCH THROW模式, 请简单说说你理解的lua的异常机制.
- lua在TRY时, 并不会直接CATCH, 而是会设置错误码
- 可以在保护模式结束后, 根据错误码来分析判断错误类别, 进而采取相应补救措施

#### 请编写模拟一下lua的异常处理代码吧. 提示, 有THROW宏, THROW宏, 以及实际的luaD_throw()函数#define TRY(L, c, a) if(setjmp((c)->b) == 0){a}#define THROW(c) longjmp((c)->b, 1) 算了, 还是直接给你吧
~~~C
#define TRY(L, c, a) if(setjmp((c)->b) == 0){a}
#define THROW(c) longjmp((c)->b, 1)

void luaD_throw(lua_State* L, int errcode){
  if(L->errjmp){
    L->errjmp->statuc = errcode;
    THROW(L->errjmp);
  } else {
    global_State* g = L->g;
    if(g->panic){
      g->panic(L);
    }
    abort();
  }
}
~~~

#### 在保护模式下执行的代码, 会统计旧的ncalls, 执行完TRY后又会赋值给ncalls, 请问为什么要这么做呢?
- 执行TRY, 内部就会ncalls++(执行函数次数)
- 如果TRY内部没有THROW, 那么这样确实"多余", 因为后续会--,保持平衡
- 如果内部THROW了, 那么就不会--了, 这样回到TRY, 并继续执行, 此时就会有保底机制, 将旧的ncalls赋值给当前L, 仿佛什么都没有发生

#### 模仿lua源码, 实现一下luaD_rawrunprotected()代码吧. 提示: TYR(L, c, a) if(setjmp(c->b) == 0){a}, int luaD_rawrunprotected(lua_State* L, Pfunc f, void* ud), 可以参考[[202605152044-lua源码-链表相关#lua的异常设计是通过链表, 来传递函数栈信息的. 这里请模拟一下dfs链式链接的代码吧. 提示, 需要有lua_longjmp结构体和lua_State结构体, 根据需要选择对应属性. 代码逻辑自己设计.|dfs]]
~~~C
#define TRY(L, c, a) if(setjmp(c->b) == 0) {a}

int luaD_rawrunprotected(lua_State* L, Pfunc f, void* ud){
  int old_ncalls = L->ncalls;
  lua_longjmp lj;
  lj.previous = L->errjmp;
  lj.status = LUA_OK;

  L->errjmp = &lj;

  TRY(L, L->errjmp, f(L, ud););

  L->errjmp = lj.previous;
  L->ncalls = old_ncalls;

  return lj.status;
}
~~~
