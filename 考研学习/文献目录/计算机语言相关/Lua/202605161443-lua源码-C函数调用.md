# Anki
#### 自己回想一下, 整个C函数调用执行流程是什么(从pcall()开始). 因为这里的函数我们现在不要求写, 所以期望你能够好好地分析思考一下具体的函数执行流程
- 首先, 用户侧执行pcall函数
- 然后会在保护模式下执行call函数
- call函数本身只是统计栈内函数调用次数的, 接下来会执行precall()
- precall()是主要运行逻辑
	- 内部会检查栈checkstack
		- checkstack会在栈不足时growstack
	- 生成函数信息ci
		- nextci函数
	- 执行函数
		- 实际调用
	- 分析函数返回值
		- poscall()

#### lua是如何统计栈内函数调用次数的?
- 首先lua_State有参数ncalls
- 其次对于指定的函数, 会包装一层luaD_call()函数, 专门用于改变ncalls的
- 最后, 会在保护模式下运行luaD_call(), 并且能够回溯old_ncalls, 即使函数运行崩溃, 也能正常计数
