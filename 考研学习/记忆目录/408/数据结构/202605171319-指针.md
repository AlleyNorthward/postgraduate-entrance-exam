# 描述
- 思考了一下, 这里其实是借鉴了线性代数中的[[202605111345-维数|维度]]的思想
- 所有的一切抽象, 都是为了更好地去解释, 只要逻辑说得通, 再复杂的抽象也说得过去
# 同维-高维
- 维度 = **声明时**\*的数量, 例如
~~~
A        A      0维对象
A*       A*     1维对象
A**      A**    2维对象
A***     A***   3维对象
A****    A****  4维对象
1维之后的对象统称为指针, 高维对象则是相对而言的描述, 指针都可以成为高维对象(例如1维对象相对于0维对象是高维, 但是相对于2为对象是低维)
~~~
- 高维(一般维度相差一维)可以控制低维行为
# 特殊操作
- **降维**
	- 使用对象时(注意, 不是声明对象), 如果添加了\*, 则是降维操作
	- 一般对左值操作
- **升维**
	- 使用对象时, 如果添加了&,则是升维操作
	- 一般对右值操作(纯C中)
# 赋值(copy)与链接
- 同维之间的=, 属于**赋值**
- 高维之间的=, 属于**链接**
	- 链接是特殊的赋值
	- 赋值更强调0维之间的
	- 链接更强调高维之间的
# 举例
~~~C
int a = 10;
int *p = &a; // a被升维了, 这里是高维之间的链接
int **pp = &p;

int *b;
b = p; //高维链接操作, b指向了p指向的低维对象a

int c = *b // b被降维了, 此时就相当于直接操作对象a了

int **d;
d = pp //高维链接操作, d指向了pp指向的那个低维对象p

int *e = *pp //pp被降维了, 此时是相当于直接操作对象p了, 那么这就是同维对象之间的链接(非0维), e会指向p指向的的那个低维对象, 也就是a

int f = **pp//pp被降了两维, 那么实际上就是同维度之间的赋值, 也就是f=a
~~~

- 赋值
