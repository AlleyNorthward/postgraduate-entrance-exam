核心一句话：

**词法分析把源码变成 token；语法分析把 token 按“语法规则”组合成 AST 树。**

你的代码在 [parser.py](<F:\0Embedded1\code\Parser Helper Functions (Exercise)_pinky\pinky\parser.py>) 里做的就是这件事。

**这些函数名是什么意思**

`parse`：解析入口  
从这里开始分析整个表达式，最后返回 AST。

`expr`：expression，表达式  
处理 `+` 和 `-`，例如：

```text
1 + 2 - 3
```

`term`：项  
处理 `*` 和 `/`，例如：

```text
2 * 3 / 4
```

它比 `expr` 更低一层，所以乘除优先级高于加减。

`factor`：因子  
现在只是转发到 `unary()`。它目前看起来没什么用，但通常是为了以后扩展，比如 `%`、`^`、位运算等。

`unary`：一元表达式  
处理只有一个操作数的运算，比如：

```text
-3
+5
~x
```

这里的 `~` 是 `TOK_NOT`。

`primary`：最基础表达式  
处理最底层、最不可再拆的东西：

```text
整数
浮点数
括号表达式
```

比如：

```text
123
3.14
(1 + 2)
```

所以从名字上看，它们是一层一层的：

```text
parse
  expr       加减
    term     乘除
      factor 预留层
        unary 一元 + - ~
          primary 数字 / 括号
```

**为什么要分这么多层**

因为语法分析要解决两个问题：

1. 这个 token 序列是否合法？
2. 如果合法，应该组成什么结构？

比如：

```text
1 + 2 * 3
```

人知道结果应该是：

```text
1 + (2 * 3)
```

而不是：

```text
(1 + 2) * 3
```

代码靠函数层级表达优先级：

```text
expr 负责 +
term 负责 *
```

`expr()` 遇到 `+` 后，会让右边先调用 `term()`。而 `term()` 会把 `2 * 3` 整体解析完，再返回给 `expr()`。

所以 AST 会变成：

```text
BinOp('+',
  Integer(1),
  BinOp('*', Integer(2), Integer(3))
)
```

这就是“乘法比加法更深”，也就是优先级更高。

**语法规则怎么看**

代码注释里的这些就是 BNF/文法：

```python
# <primary> ::= <integer> | <float> | '(' <expr> ')'
```

意思是：

```text
primary 可以是 integer
也可以是 float
也可以是 括号包起来的 expr
```

这个 `|` 表示“或者”。

再看：

```python
# <expr> ::= <term> ( ('+'|'-') <term> )*
```

意思是：

```text
expr 至少有一个 term
后面可以跟 0 个或多个 + term 或 - term
```

所以这些都合法：

```text
1
1 + 2
1 + 2 - 3
```

`*` 表示“重复零次或多次”。

**语法分析流程怎么走**

假设源码是：

```text
1 + 2 * 3
```

词法分析后大概是：

```text
TOK_INTEGER("1")
TOK_PLUS("+")
TOK_INTEGER("2")
TOK_STAR("*")
TOK_INTEGER("3")
```

然后 parser 开始：

```text
parse()
  expr()
    term()
      factor()
        unary()
          primary()
            看到 1，返回 Integer(1)

    expr 看到 +
    右边继续 term()

    term()
      factor -> unary -> primary
        看到 2，返回 Integer(2)

      term 看到 *
      右边继续 factor -> unary -> primary
        看到 3，返回 Integer(3)

      term 组装 BinOp('*', Integer(2), Integer(3))

    expr 组装 BinOp('+', Integer(1), BinOp('*', Integer(2), Integer(3)))
```

最后 `parse()` 返回整棵 AST。

重点是：**调用时从大规则往小规则走，返回时从小节点往大节点组树。**

你可以记成：

```text
向下：找最小语法单位
向上：组装 AST 节点
```

**primary 为什么又能调用 expr**

这里最容易绕：

```python
if self.match(TOK_LPAREN):
  expr = self.expr()
```

因为括号里可以放一个完整表达式：

```text
(1 + 2 * 3)
```

所以 `primary()` 遇到 `(` 后，会重新启动一轮 `expr()`，把括号里的东西完整解析完，然后要求后面必须有 `)`。

这就是递归下降里的“递归”。

**AST 模型是什么**

在 [model.py](<F:\0Embedded1\code\Parser Helper Functions (Exercise)_pinky\pinky\model.py>) 里：

```python
Integer
Float
UnOp
BinOp
Grouping
```

这些不是 token，而是 AST 节点。

token 是词法层面的：

```text
TOK_INTEGER("123")
TOK_PLUS("+")
```

AST 是语义结构层面的：

```text
BinOp('+', Integer(1), Integer(2))
```

也就是说：

```text
token 只说“我看到了什么符号”
AST 说明“这些符号组成了什么结构”
```

后面解释器、代码生成器、优化器，都不会直接看原始字符串，而是看 AST。

**pass 那些 helper 的概念**

你说 pass 不用写，我只说意思：

`advance`：消费当前 token，移动到下一个。  
`peek`：看当前 token，但不消费。  
`match`：如果当前 token 是某类型，就消费它并返回成功；否则失败。  
`previous_token`：拿刚刚消费掉的 token。  
`expect`：强制要求某 token 出现，否则报语法错误。  
`is_next`：判断当前/下一个 token 是否是某类型。

这些 helper 是 parser 的“读 token 工具”，真正的语法逻辑在 `expr/term/unary/primary`。

**你现在最该抓住的理解**

不要先盯着每一行代码。先抓这个模型：

```text
expr    管 + -
term    管 * /
unary   管前缀 + - ~
primary 管数字和括号
```

然后再记住：

```text
parse 从 expr 开始
expr 调 term
term 调 factor
factor 调 unary
unary 调 primary
primary 可能又因为括号调回 expr
```

这就是递归下降语法分析。它把 BNF 文法规则，一条一条翻译成 Python 函数。
